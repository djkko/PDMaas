## 1. 项目介绍 | Intro

PDMaas是一款优秀的国产数据建模工具，所有操作系统都能用，主打轻便好看效率高。它能帮你：

1. 拖拽画数据表和ER关系图，概念/逻辑/物理三层建模  
2. 把流程图、思维导图整合到一起看  
3. 主流数据库（如MySQL、PostgreSQL）、国产数据库、大数据MPP库全兼容  
4. 有标准字段库和枚举代码库  
5. 自动生成Word、HTML、Markdown文档和Java/Python等代码  
6. 模型和数据库双向同步，自动生成DDL差异脚本  
7. 支持多版本管理，便捷追溯模型历史变更  

- **核心定位**：社区开源版（CE）面向开发者，支持全流程数据模型设计与管理。   
- **适用场景**：开发者、技术爱好者

### 1.1 数据库支持
- 主流数据库：MySQL、PostgreSQL、Oracle、SQLServer、DB2、SQLite
- 国产信创数据库：DaMeng（达梦）、KingBase（金仓）、GBase（南大通用）、ShenTong（神州通用）、OceanBase（海扬）、XuguDB（虚谷）、 GaussDB（高斯DB）、HighGo（瀚高）
- 大数据平台：Hive、Doris、StarRocks、Impala、ClickHouse、MaxCompute、星环Inceptor&ArgoDB
- 待支持数据库：GoldenDB（金篆）、TiDB

### 1.2 发展历史  

2018年初，团队针对传统建模工具局限，仅用一个半月完成PDMan 1.0开发，实现产品从无到有突破；同年5月发布首个开源版本，开启开源探索之路，持续更新至2019年1月。因前期设计存在局限，升级难度增大，2019年底团队启动系统性规划设计。

2022年4月，融合PDMan核心标识与实体关系图（ER）概念的PDManer 4.0正式上线，全面承接并超越过往版本功能，迅速跻身国内开源数据库工具领先行列。

随着数据要素市场加速构建及AI技术迅猛兴起，数据的核心价值愈发凸显，企业对数据建模在大规模协作、标准化规范及在线服务化部署等方面的需求持续升级，PDManer架构逐渐难以满足需求。

2023年团队开启商业化探索，基于PDManer推出企业级协作产品PDMaas。2024年7月，第二代PDMaas发布，完成架构全面升级，**确立PDMaas为统一品牌，划分版本**：
- **PDMaas-CE (社区单机版)**：免费开源社区版，适用于技术爱好者；
- **PDMaas-Pro (高级单机版)**：高级功能单机版，满足专业个人需求；
- **PDMaas-EE (企业级Web版)**：功能全集 + 企业级协作平台，面向企业及团队用户。

此次升级完整承接原PDManer 98%以上功能，成功实现从开源社区工具到企业级全方位数据建模解决方案的转型。
**PDMaas将成为本产品长期使用的唯一名称，不再进行任何变更。**

如同红军→八路军→解放军的历史演进，正是PDMan→PDManer→PDMaas产品迭代的最佳类比：不同历史阶段承担不同使命，核心团队始终如一的坚守，以及能力与定位的持续性升级，共同推动着产品不断进化。

## 2. 快速上手 | Get Started  
### 2.1 环境依赖
- 软件：node v16.19.0+ ; npm v8.19.3+
- 常见依赖问题：https://zhuanlan.zhihu.com/p/248742896

### 2.2 源码编译步骤
1. 克隆代码到本地： ```clone: git clone https://gitee.com/yonsum/PDMaas.git```
2. 安装依赖：```npm i --force```
3. 编译打包(根据自己平台选择）：
    - 一键打包所有操作系统版本：```npm run package-all```
    - 打包 mac: ```npm run package-mac```
    - 打包 linux: ```npm run package-linux```
    - 打包windows: ```npm run package-win```
4. 编译成功后：会在项目的根目录下的dist目录生成对应的安装包
5. 本地启动：```npm run start```

## 3. 版本对比 | Editions Comparison  
### 3.1 各版本定位说明
**1. 社区版（CE）「桌面客户端」**
- 授权模式：功能完整的开源版本，采用 AGPL-3.0 协议，开放全部源代码。
- 核心特点：延续开源基因，为基础建模需求提供免费支持。
- 适用群体：开发者、技术爱好者

**2. 专业版（Pro）「桌面客户端」**
- 授权模式：核心功能免费开放，高级功能需通过购买高级版扩展插件获取。
- 核心特点：涵盖单机版核心功能全集，满足专业级单机建模日常需求；支持 AI 智能建模、高级自动化生成等进阶功能，兼顾核心需求与升级空间。
- 适用群体：对单机功能要求较高的专业个人用户、自由职业者及单人使用者。

**3. 企业版（EE）「Web 应用」**
- 授权模式：功能全集，采用商业授权模式。
- 核心特点：基于浏览器的企业级解决方案，支持企业级集中管理、权限划分、团队分工、实时协作、分支版本管理、批注审阅、系统集成及企业服务等，提供便捷访问体验。
- 适用群体：小型团队、中小型企业，及大型企业的部门或分子公司。

## 4. 功能架构 | Architecture  
### 4.1 功能架构全景图  
![输入图片说明](public/readme/all-func-struct.png)
PDMaas以协作为核心，围绕数据建模提供一体化解决方案，覆盖从设计到实施的全过程。支持正向与逆向建模，确保数据模型高效设计并持续演进。平台无缝对接企业其他研发系统，实现数据、标准和流程的协同管理，帮助企业建立统一的数据模型，推动数据标准的规范落地，全面提升数据的准确性、完整性和一致性。
PDMaas全面支持国产信创环境，兼容超过20款主流及国产数据库如达梦、金仓等，保障企业数据建模的安全性与自主可控。

**PDMaas社区版**：包含数据标准化、数据模型设计器、支持的数据库的部分（排除建模协同部分）。系统管理以及团队管理涉及企业管理，这部分功能在企业版中有完整的支持。

### 4.2 技术架构
PDMaas企业版技术选型采用主流的开发技术及框架，基于庞大的用户生态，更有利于软件的持续维护升级。
- 后端使用：Java17+springboot3，
- 前端使用：ES+TS+Vue+React这一主流混合的前端技术，采用用微前端技术将他们粘合为整体。
- 所有有“技术组件”均通过全栈信创验证，确保从底层开始就支持信创环境。
![输入图片说明](public/readme/tech-struct.png)
**PDMaas社区版**：涉及技术请查阅架构图的右上“模型设计器”部分即可，模型设计器能够包含PDMaas社区版的所有核心功能，仅不含企业协作以及Java服务调用部分。

### 4.3 部署架构
![输入图片说明](public/readme/deploy-struct.png)
**PDMaas社区版**：社区版以及Pro（专业版）均是客户端单机工具，不需要像企业版BS架构那样，安装即可使用，不需要复杂的部署。

### 4.4 三个版本的架构关联说明  
PDMaas 核心引擎全面统一，社区版、专业版、企业版均基于同一技术基座构建，所有用户均可享受与企业级版本同代的核心引擎。在技术底层能力上，新引擎实现了三方面的提升：
1. 性能显著提升：针对大规模数据模型（支持 5000 张以上表结构，较此前最高 800 张的上限大幅提升）的打开操作，以及企业版的并行协作场景，可支持数十万级字段量的数据模型建模设计。
2. 信创全面适配：深度兼容国产操作系统（如麒麟、统信），支持国产数据库（如达梦、金仓、南大通用、神通、海扬数据库等超过23款主流及国产数据库）。
3. 灵活扩展支持：支持用户自行添加新的数据库品牌，同时支持自定义实体级及字段级属性，满足更多技术或管理层面的属性需求。
